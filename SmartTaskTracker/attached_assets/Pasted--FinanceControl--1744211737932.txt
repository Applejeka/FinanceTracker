ехническое задание на разработку приложения "FinanceControl" для анализа и оптимизации личных расходов
1. Введение
1.1 Наименование программного продукта

"FinanceControl" - приложение для учета и анализа личных финансов.
1.2 Назначение разработки

Создание удобного инструмента для контроля доходов и расходов с возможностью визуализации данных и генерации отчетов.
2. Требования к функциональным характеристикам
2.1 Основные функции:

    Учет доходов и расходов

    Категоризация финансовых операций

    Визуализация данных (графики, диаграммы)

    Генерация отчетов

    Управление транзакциями (добавление, редактирование, удаление)

2.2 Требования к интерфейсу:

    Главное окно с навигационным меню

    Форма добавления транзакций

    Просмотр истории операций

    Раздел с аналитикой и визуализацией данных

    Настройки приложения

3. Технические требования
3.1 Используемые технологии:

    Язык программирования: Python 3.10+

    Графическая библиотека: PyQt6

    База данных: SQLite3

    Дополнительные библиотеки: matplotlib, pandas

3.2 Системные требования:

    Операционная система: Windows 10/11

    Процессор: 2 ядра, 1.6 GHz

    Оперативная память: 4 GB

    Дисковое пространство: 50 MB

4. Описание модулей приложения
4.1 Основные модули:

    main.py - точка входа в приложение

    database.py - работа с базой данных SQLite

    models.py - модели данных (транзакции, категории)

    views.py - графический интерфейс (PyQt)

    reports.py - генерация отчетов и визуализация данных

    settings.py - настройки приложения

4.2 Файловая структура:
Copy

FinanceControl/
├── main.py
├── database.py
├── models.py
├── views.py
├── reports.py
├── settings.py
├── static/
│   ├── styles.css

    ├── manual.pdf

5. Требования к базе данных
5.1 Структура БД:

    Таблица transactions:

        id (INTEGER, PRIMARY KEY)

        amount (REAL)

        category_id (INTEGER, FOREIGN KEY)

        date (TEXT)

        description (TEXT)

        type (TEXT) - income/expense

    Таблица categories:

        id (INTEGER, PRIMARY KEY)

        name (TEXT)

        color (TEXT)

6. Графический интерфейс
6.1 Основные окна:

    Главное окно:

        Меню навигации

        Краткая статистика

        Быстрый доступ к основным функциям

    Окно добавления транзакции:

        Поля: сумма, категория, дата, описание

        Кнопки: сохранить, отменить

    Окно истории транзакций:

        Таблица с операциями

        Фильтры по дате и категориям

        Кнопки: редактировать, удалить

    Окно аналитики:

        Диаграммы расходов по категориям

        Графики динамики доходов/расходов

        Кнопка генерации отчета
